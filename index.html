SABER HACER 1: Reporte del tipo de empaquetado y la plataforma digital de distribución (EXTERNA)

1. Tipo de empaquetado

Android

  • APK (Android Package Kit): formato ideal para distribución externa, permite instalar la app mediante archivo.
  • AAB: no se usa en distribución externa, requiere Google Play.

iOS

La distribución externa es limitada debido a las restricciones de Apple. Solo se permite mediante:

  • TestFlight
  • MDM empresarial
  • Ad Hoc Distribution

2. Plataforma digital de distribución EXTERNA

a) Sitio web o servidor propio

  • Descarga del APK desde página web.
  • Google Drive / Dropbox.
  • Servidor privado o institucional.

b) Servicios de distribución externa

  • Firebase App Distribution
  • Diawi
  • App Center

c) Distribución empresarial

  • Sistemas MDM
  • Instalación mediante código QR
  • Enlace directo controlado internamente
SABER HACER 2: Reporte Digital – Canales de distribución y herramientas de despliegue (EXTERNA)

1. Canales de distribución EXTERNA

a) Descarga directa

  • Archivo APK desde página web
  • Enlaces de nube (Drive, Mega, MediaFire)
  • Código QR

b) Distribución interna

  • Servidores locales
  • Sistemas MDM
  • Redes internas

c) Plataformas externas de pruebas

  • Firebase App Distribution
  • App Center
  • Diawi

2. Herramientas de despliegue

Android

  • Android Studio (para generar APK)
  • Firebase App Distribution
  • App Center
  • Diawi

iOS

  • Xcode
  • TestFlight
  • MDM o certificados Ad Hoc
SABER HACER 3: Reporte Digital – Requisitos y políticas para distribución externa

1. Requisitos técnicos

  • Permitir instalación desde “fuentes desconocidas”.
  • APK firmado digitalmente.
  • Uso correcto de permisos.
  • Optimización del archivo APK.

2. Requisitos de seguridad

  • Proteger API Keys y archivos sensibles.
  • Usar HTTPS en toda comunicación.
  • Evitar modificación del APK mediante firma segura.

3. Requisitos legales

  • Incluir Política de Privacidad accesible.
  • Informar sobre uso de permisos (ubicación, cámara, contactos).
  • Solicitar consentimiento si se recaban datos.
SABER HACER 4: Reporte Digital – Publicación de la aplicación (EXTERNA)

1. Preparación previa

  • Generar APK firmado.
  • Probar en varios dispositivos.
  • Reunir: nombre, versión, icono, capturas y APK.

2. Métodos de publicación externa

a) Página web / servidor

  • Subir APK.
  • Generar enlace de descarga.
  • Compartir enlace con usuarios.

b) Enlace directo

  • Google Drive
  • Mega
  • Dropbox

c) Código QR

  • Generar un QR que dirija al APK.

d) Plataformas externas

  • Firebase App Distribution
  • App Center
  • Diawi

3. Instalación por parte del usuario

  • Activar “Fuentes desconocidas”.
  • Ejecutar el APK.
  • Aceptar permisos.

4. Actualizaciones

  • Subir nuevo APK a la misma ruta.
  • Notificar a los usuarios del nuevo enlace.
